## 2025-02-14 - Pipe Deadlock in IO Reading
**Vulnerability:** In both `src/autobot/tools/sandbox.cr` and `src/autobot/tools/exec.cr`, the `read_limited_output` function reads child process output up to a certain maximum size and then breaks the loop immediately, leaving the unread data in the OS pipe buffer. If the child process generates more output, it will eventually fill up the pipe buffer and block indefinitely on a `write()` system call, waiting for the parent process to read data (which it never will since it exited the loop). This causes a Denial of Service (DoS) where the child process hangs forever.
**Learning:** Breaking early from an IO stream read loop connected to a child process pipe causes a pipe deadlock. The child process hangs if the pipe buffer fills up because the parent is no longer draining it.
**Prevention:** When reading from an IO stream with a maximum size limit, use a bounded loop that continues reading and discarding the excess data until EOF instead of breaking early. This ensures the pipe remains drained, allowing the child process to complete without hanging.
